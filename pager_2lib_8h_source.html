66 // Pager mode.
67 // There are 10 code paths that lead to mutt_pager() invocation:
68 //
69 // 1. mutt_index_menu -> mutt_display_message -> mutt_pager
70 //
71 // This path always results in mailbox and email set,
72 // the rest is unset - Body, fp.
73 // This invocation can be identified by IsEmail macro.
74 // The intent is to display an email message
75 //
76 // 2. mutt_view_attachment -> mutt_do_pager -> mutt_pager
77 //
78 // this path always results in email, body, ctx set
79 // this invocation can be identified by one of the two macros
80 // - IsAttach (viewing a regular attachment)
81 // - IsMsgAttach (viewing nested email)
82 //
83 // IsMsgAttach has extra->body->email set
84 // extra->email != extra->body->email
85 // the former is the message that contains the latter message as attachment
86 //
87 // NB. extra->email->body->email seems to be always NULL
88 //
89 // 3. The following 8 invocations are similar, because they all call
90 // mutt_do_page with info = NULL
91 //
92 // And so it results in mailbox, body, fp set to NULL.
93 // The intent is to show user some text that is not
94 // directly related to viewing emails,
95 // e.g. help, log messages,gpg key selection etc.
96 //
97 // No macro identifies these invocations
98 //
99 // mutt_index_menu -> mutt_do_pager -> mutt_pager
100 // mutt_help -> mutt_do_pager -> mutt_pager
101 // icmd_bind -> mutt_do_pager -> mutt_pager
102 // icmd_set -> mutt_do_pager -> mutt_pager
103 // icmd_version -> mutt_do_pager -> mutt_pager
104 // dlg_select_pgp_key -> mutt_do_pager -> mutt_pager
105 // verify_key -> mutt_do_pager -> mutt_pager
106 // mutt_invoke_sendmail -> mutt_do_pager -> mutt_pager
107 //
108 //
109 // - IsAttach(pager) (pager && (pager)->body)
110 // - IsMsgAttach(pager)
111 // (pager && (pager)->fp && (pager)->body && (pager)->body->email)
112 // - IsEmail(pager) (pager && (pager)->email && !(pager)->body)
113 // See nice infographic here:
114 // https://gist.github.com/flatcap/044ecbd2498c65ea9a85099ef317509a
